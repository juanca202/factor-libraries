# ADR-003: Use of Tailwind CSS for Utility Classes and Component Creation

**📖 [English](./ADR-003.md) | [Español](../../es/adr/ADR-003.md)**

**Status:** Accepted  
**Date:** 2025  
**Deciders:** Architecture Team

## Context

CSS management in large applications can become challenging:
- **Dead CSS:** Unused styles accumulate over time, increasing bundle size
- **Inconsistency:** Different developers use different spacing, colors, and sizing values
- **Maintenance:** Updating styles across multiple files is time-consuming
- **Performance:** Large CSS files can impact initial load time
- **Developer Experience:** Writing custom CSS for every component slows development

Utility-first CSS frameworks like Tailwind CSS address these issues by providing a comprehensive set of utility classes that can be composed to build any design.

## Decision

We will use **Tailwind CSS** as the primary CSS framework for all utility classes (layout, spacing, typography, colors, etc.), while reserving custom CSS classes with the `ft-` prefix only for component-specific styles that follow the BEM methodology.

## Structure

### Tailwind CSS Usage

Tailwind CSS should be used for:
- **Layout:** `flex`, `grid`, `container`, `block`, `inline-block`
- **Spacing:** `p-4`, `m-2`, `gap-6`, `space-x-4`
- **Typography:** `text-lg`, `font-bold`, `text-center`, `leading-tight`
- **Colors:** `bg-blue-500`, `text-gray-700`, `border-red-300`
- **Sizing:** `w-full`, `h-screen`, `max-w-4xl`
- **Responsive Design:** `md:flex`, `lg:grid-cols-3`, `sm:text-xl`
- **States:** `hover:bg-blue-600`, `focus:ring-2`, `disabled:opacity-50`

### Custom CSS with BEM

Custom CSS classes with the `ft-` prefix should only be used for:
- Component-specific styles that cannot be achieved with Tailwind utilities
- Complex animations or transitions
- Component-specific theming
- Styles that need to be reused across multiple instances of the same component

**Naming Convention:** `ft-{component-name}__{element}--{modifier}`

## Examples

### Example 1: Using Tailwind for Layout and Spacing

```html
<!-- ✅ Correct - Using Tailwind utilities -->
<div class="flex flex-col gap-4 p-6 bg-white rounded-lg shadow-md">
  <h2 class="text-2xl font-bold text-gray-900 mb-4">Sales List</h2>
  <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
    <app-sale-item [sale]="sale" />
  </div>
</div>

<!-- ❌ Incorrect - Custom CSS for common utilities -->
<div class="sales-container">
  <h2 class="sales-title">Sales List</h2>
  <div class="sales-grid">
    <app-sale-item [sale]="sale" />
  </div>
</div>

</style>
```

### Example 2: Combining Tailwind with Custom BEM Classes

```html
<!-- ✅ Correct - Tailwind for layout, custom class for component-specific styling -->
<div class="flex items-center gap-4 p-4 bg-white rounded-lg">
  <div class="ft-user-card__avatar">
    <img [src]="user.avatar" [alt]="user.name" />
  </div>
  <div class="ft-user-card__content">
    <h3 class="text-lg font-semibold">{{ user.name }}</h3>
    <p class="text-gray-600">{{ user.email }}</p>
  </div>
</div>
```

```scss
// Component-specific styles using BEM
.ft-user-card__avatar {
  position: relative;
  width: 48px;
  height: 48px;
  border-radius: 50%;
  overflow: hidden;
  
  // Complex animation that Tailwind can't easily handle
  &::after {
    content: '';
    position: absolute;
    inset: 0;
    border: 2px solid transparent;
    border-radius: 50%;
    transition: border-color 0.3s ease;
  }
  
  &:hover::after {
    border-color: var(--primary-color);
  }
}

.ft-user-card__content {
  flex: 1;
  min-width: 0; // Text truncation support
}
```

### Example 4: Component with Tailwind and Custom Styles

```typescript
@Component({
  selector: 'ft-button',
  template: `
    <button
      [class]="buttonClasses()"
      [disabled]="disabled()"
      (click)="handleClick()"
    >
      <ft-icon *ngIf="icon()" [name]="icon()" class="ft-button__icon" />
      <ng-content />
    </button>
  `,
  styles: [`
    .ft-button__icon {
      // Component-specific icon positioning
      margin-right: 0.5rem;
      vertical-align: middle;
    }
    
    .ft-button--loading {
      // Loading state animation
      position: relative;
      color: transparent;
      
      &::after {
        content: '';
        position: absolute;
        inset: 0;
        border: 2px solid currentColor;
        border-top-color: transparent;
        border-radius: 50%;
        animation: spin 0.6s linear infinite;
      }
    }
  `]
})
export class ButtonComponent {
  variant = input<'primary' | 'secondary' | 'danger'>('primary');
  size = input<'sm' | 'md' | 'lg'>('md');
  disabled = input<boolean>(false);
  loading = input<boolean>(false);
  icon = input<string | undefined>(undefined);
  
  buttonClasses = computed(() => {
    const base = 'inline-flex items-center justify-center font-medium rounded-lg transition-colors';
    const variants = {
      primary: 'bg-blue-600 text-white hover:bg-blue-700',
      secondary: 'bg-gray-200 text-gray-900 hover:bg-gray-300',
      danger: 'bg-red-600 text-white hover:bg-red-700',
    };
    const sizes = {
      sm: 'px-3 py-1.5 text-sm',
      md: 'px-4 py-2 text-base',
      lg: 'px-6 py-3 text-lg',
    };
    const state = this.disabled() ? 'opacity-50 cursor-not-allowed' : 'cursor-pointer';
    const loading = this.loading() ? 'ft-button--loading' : '';
    
    return `${base} ${variants[this.variant()]} ${sizes[this.size()]} ${state} ${loading}`;
  });
}
```

## Configuration

Tailwind CSS is configured in `tailwind.config.js` with:
- Custom theme extensions (colors, spacing, typography)
- Project-specific breakpoints
- Custom utility classes if needed

```javascript
// tailwind.config.js example
module.exports = {
  content: [
    "./src/**/*.{html,ts}",
  ],
  theme: {
    extend: {
      colors: {
        primary: {
          DEFAULT: '#3b82f6',
          dark: '#2563eb',
        },
      },
    },
  },
  plugins: [],
}
```

## Consequences

### Positive

- **Faster Development:** Pre-built utility classes speed up UI development
- **Consistency:** Standardized spacing, colors, and sizing across the application
- **Smaller Bundle Size:** Only used CSS is included in the final bundle (with JIT mode)
- **Maintainability:** No need to maintain custom CSS for common patterns
- **Responsive Design:** Built-in responsive utilities make mobile-first development easier
- **Developer Experience:** Less context switching between HTML and CSS files
- **Design System:** Tailwind provides a solid foundation for a design system

### Negative

- **Learning Curve:** Team members need to learn Tailwind's utility class names
- **HTML Verbosity:** HTML can become more verbose with many utility classes
- **Custom Styles:** Some complex designs may still require custom CSS
- **Purge Configuration:** Need to ensure Tailwind correctly detects all used classes

### Mitigation

- Comprehensive Tailwind documentation and cheat sheets
- IDE extensions for Tailwind autocomplete
- Code review process to ensure proper usage
- Custom CSS guidelines clearly documented
- Regular team training on Tailwind best practices

## When to Use Custom CSS

Use custom CSS with the `ft-` prefix when:
1. **Complex Animations:** Multi-step animations that are difficult with Tailwind
2. **Component-Specific Logic:** Styles that depend on component state in complex ways
3. **Third-Party Integration:** Styling third-party components that can't use Tailwind
4. **Performance:** Critical styles that need to be optimized separately
5. **Reusability:** Component styles that will be reused across the application
