# ADR-013: Form Layout and Structure

**📖 [English](./ADR-013.md) | [Español](../../es/adr/ADR-013.md)**

**Status:** Accepted  
**Date:** 2025  
**Deciders:** Architecture Team

## Context

Form layout and structure directly impact user experience and code maintainability. Without standardized layout patterns:

- Forms have inconsistent spacing and alignment
- Responsive behavior is unpredictable
- Code duplication increases when similar layouts are recreated
- Visual hierarchy is unclear
- Maintenance becomes difficult when layout patterns vary

We need a consistent approach that:
- Provides predictable, responsive form layouts
- Ensures visual consistency across the application
- Makes forms easy to maintain and modify
- Works seamlessly with Angular Material components
- Follows modern CSS Grid best practices

## Decision

We will use **CSS Grid** with Tailwind CSS utility classes for all form layouts, following these standardized patterns:

1. **Grid Container:** All form fields must be contained within a `div` with `grid` class
2. **Column Configuration:** Use `grid-cols-{n}` to define the number of columns based on content
3. **Full-Width Fields:** Fields spanning full width use `col-span-{n}` where `n` matches the number of columns
4. **Spacing:** Standardized spacing using `gap-x-4 gap-y-2`
5. **Form Fields:** Simple `mat-form-field` without appearance or width classes

## Implementation

### Grid Container Structure

All form fields must be wrapped in a grid container:

```html
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <!-- Form fields here -->
</div>
```

### Column Configuration

Choose the number of columns based on the form's content and design requirements:

- **2 columns:** Most common for forms with pairs of related fields (e.g., first name + last name)
- **3 columns:** For forms with triplets of related fields
- **1 column:** For simple forms or when all fields should be full width

```html
<!-- 2-column grid (most common) -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>...</mat-form-field>
  <mat-form-field>...</mat-form-field>
</div>

<!-- 3-column grid -->
<div class="grid grid-cols-3 gap-y-2 gap-x-4">
  <mat-form-field>...</mat-form-field>
  <mat-form-field>...</mat-form-field>
  <mat-form-field>...</mat-form-field>
</div>

<!-- 1-column grid (full width fields) -->
<div class="grid grid-cols-1 gap-y-2 gap-x-4">
  <mat-form-field>...</mat-form-field>
</div>
```

### Full-Width Fields

Fields that should span the full width of the grid use `col-span-{n}` where `n` matches the number of columns:

```html
<!-- 2-column grid with full-width fields -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>
    <mat-label i18n>First name</mat-label>
    <input matInput formControlName="firstName" />
  </mat-form-field>
  <mat-form-field>
    <mat-label i18n>Last name</mat-label>
    <input matInput formControlName="lastName" />
  </mat-form-field>
  <mat-form-field class="col-span-2">
    <mat-label i18n>Email</mat-label>
    <input matInput type="email" formControlName="email" />
  </mat-form-field>
  <mat-form-field class="col-span-2">
    <mat-label i18n>Phone</mat-label>
    <input matInput formControlName="phone" />
  </mat-form-field>
</div>
```

### Spacing

Standardized spacing ensures consistent visual rhythm:

- **Horizontal spacing:** `gap-x-4` (1rem / 16px)
- **Vertical spacing:** `gap-y-2` (0.5rem / 8px)

```html
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <!-- Fields automatically spaced -->
</div>
```

### Form Field Structure

Form fields should be simple and clean:

- **No appearance attribute:** Do not use `appearance="outline"` or other appearance variants
- **No width classes:** Do not use `class="ft-w-100"` or similar width utilities
- **Simple structure:** Let the grid handle layout and spacing

```html
<!-- ✅ Correct -->
<mat-form-field>
  <mat-label i18n>Field Label</mat-label>
  <input matInput formControlName="fieldName" />
  @if (form.get('fieldName')?.invalid && form.get('fieldName')?.touched) {
    <mat-error>{{ form.get('fieldName') | errorMessage }}</mat-error>
  }
</mat-form-field>

<!-- ❌ Incorrect -->
<mat-form-field appearance="outline" class="ft-w-100">
  <mat-label i18n>Field Label</mat-label>
  <input matInput formControlName="fieldName" />
</mat-form-field>
```

## Complete Form Example

```html
<form [formGroup]="form" (ngSubmit)="onSubmit()">
  <div mat-dialog-content>
    <div class="ft-container">
      <div class="grid grid-cols-2 gap-y-2 gap-x-4">
        <mat-form-field>
          <mat-label i18n>First name</mat-label>
          <input matInput formControlName="firstName" />
          @if (form.get('firstName')?.invalid && form.get('firstName')?.touched) {
            <mat-error>{{ form.get('firstName') | errorMessage }}</mat-error>
          }
        </mat-form-field>
        
        <mat-form-field>
          <mat-label i18n>Last name</mat-label>
          <input matInput formControlName="lastName" />
          @if (form.get('lastName')?.invalid && form.get('lastName')?.touched) {
            <mat-error>{{ form.get('lastName') | errorMessage }}</mat-error>
          }
        </mat-form-field>
        
        <mat-form-field class="col-span-2">
          <mat-label i18n>Email</mat-label>
          <input matInput type="email" formControlName="email" />
          @if (form.get('email')?.invalid && form.get('email')?.touched) {
            <mat-error>{{ form.get('email') | errorMessage }}</mat-error>
          }
        </mat-form-field>
        
        <mat-form-field class="col-span-2">
          <mat-label i18n>Phone</mat-label>
          <input matInput formControlName="phone" />
          @if (form.get('phone')?.invalid && form.get('phone')?.touched) {
            <mat-error>{{ form.get('phone') | errorMessage }}</mat-error>
          }
        </mat-form-field>
      </div>
    </div>
  </div>
  
  <div mat-dialog-actions>
    <button type="button" mat-button mat-dialog-close i18n>Cancel</button>
    <button type="submit" mat-flat-button color="primary" [disabled]="form.invalid" i18n>
      Save
    </button>
  </div>

```

## Best Practices

### 1. Choose Appropriate Column Count

```html
<!-- ✅ Correct - 2 columns for name fields -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>First name</mat-form-field>
  <mat-form-field>Last name</mat-form-field>
</div>

<!-- ✅ Correct - 1 column for single field or all full-width -->
<div class="grid grid-cols-1 gap-y-2 gap-x-4">
  <mat-form-field>Description</mat-form-field>
</div>
```

### 2. Use col-span for Full-Width Fields

```html
<!-- ✅ Correct -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>Field 1</mat-form-field>
  <mat-form-field>Field 2</mat-form-field>
  <mat-form-field class="col-span-2">Full width field</mat-form-field>
</div>

<!-- ❌ Incorrect - don't use separate grid or width classes -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>Field 1</mat-form-field>
  <mat-form-field>Field 2</mat-form-field>
</div>
<div class="grid grid-cols-1">
  <mat-form-field class="ft-w-100">Full width field</mat-form-field>
</div>
```

### 3. Keep Form Fields Simple

```html
<!-- ✅ Correct - simple, clean structure -->
<mat-form-field>
  <mat-label i18n>Label</mat-label>
  <input matInput formControlName="field" />
</mat-form-field>

<!-- ❌ Incorrect - unnecessary attributes and classes -->
<mat-form-field appearance="outline" class="ft-w-100">
  <mat-label i18n>Label</mat-label>
  <input matInput formControlName="field" />
</mat-form-field>
```

### 4. Consistent Spacing

```html
<!-- ✅ Correct - always use gap-x-4 gap-y-2 -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <!-- Fields -->
</div>

<!-- ❌ Incorrect - inconsistent spacing -->
<div class="grid grid-cols-2 gap-4">
  <!-- Fields -->
</div>
```

## Responsive Considerations

For responsive forms, consider using Tailwind's responsive breakpoints:

```html
<!-- 1 column on mobile, 2 columns on larger screens -->
<div class="grid grid-cols-1 md:grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>Field 1</mat-form-field>
  <mat-form-field>Field 2</mat-form-field>
  <mat-form-field class="md:col-span-2">Full width on desktop</mat-form-field>
</div>
```


## Consequences

### Positive

- **Consistency:** All forms follow the same layout pattern
- **Maintainability:** Easy to modify and update form layouts
- **Responsiveness:** Grid system provides built-in responsive capabilities
- **Visual Harmony:** Consistent spacing creates better visual rhythm
- **Code Clarity:** Clear structure makes forms easier to read and understand
- **Flexibility:** Grid system allows for various column configurations

### Negative

- **Learning Curve:** Team members need to learn the grid pattern
- **Rigidity:** May not fit all edge cases (though grid is very flexible)
- **Tailwind Dependency:** Requires Tailwind CSS to be available

### Mitigation

- Comprehensive documentation in this ADR
- Code examples and templates
- Code review process to ensure pattern compliance
- Clear guidelines on when to use different column counts
