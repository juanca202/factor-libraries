# ADR-003: Uso de Tailwind CSS para Clases de Utilidad y Creación de Componentes

**📖 [English](../../en/adr/ADR-003.md) | [Español](./ADR-003.md)**

**Estado:** Aceptado  
**Fecha:** 2025  
**Decisores:** Equipo de Arquitectura

## Contexto

La gestión de CSS en aplicaciones grandes puede volverse desafiante:
- **CSS Muerto:** Los estilos no utilizados se acumulan con el tiempo, aumentando el tamaño del bundle
- **Inconsistencia:** Diferentes desarrolladores usan diferentes valores de espaciado, colores y tamaños
- **Mantenimiento:** Actualizar estilos en múltiples archivos consume tiempo
- **Rendimiento:** Archivos CSS grandes pueden impactar el tiempo de carga inicial
- **Experiencia del Desarrollador:** Escribir CSS personalizado para cada componente ralentiza el desarrollo

Los frameworks CSS utility-first como Tailwind CSS abordan estos problemas proporcionando un conjunto completo de clases de utilidad que pueden componerse para construir cualquier diseño.

## Decisión

Usaremos **Tailwind CSS** como el framework CSS principal para todas las clases de utilidad (layout, espaciado, tipografía, colores, etc.), reservando clases CSS personalizadas con el prefijo `ft-` solo para estilos específicos de componentes que sigan la metodología BEM.

## Estructura

### Uso de Tailwind CSS

Tailwind CSS debe usarse para:
- **Layout:** `flex`, `grid`, `container`, `block`, `inline-block`
- **Espaciado:** `p-4`, `m-2`, `gap-6`, `space-x-4`
- **Tipografía:** `text-lg`, `font-bold`, `text-center`, `leading-tight`
- **Colores:** `bg-blue-500`, `text-gray-700`, `border-red-300`
- **Tamaños:** `w-full`, `h-screen`, `max-w-4xl`
- **Diseño Responsivo:** `md:flex`, `lg:grid-cols-3`, `sm:text-xl`
- **Estados:** `hover:bg-blue-600`, `focus:ring-2`, `disabled:opacity-50`

### CSS Personalizado con BEM

Las clases CSS personalizadas con el prefijo `ft-` solo deben usarse para:
- Estilos específicos de componentes que no pueden lograrse con utilidades de Tailwind
- Animaciones o transiciones complejas
- Temas específicos de componentes
- Estilos que necesitan reutilizarse en múltiples instancias del mismo componente

**Convención de Nombres:** `ft-{component-name}__{element}--{modifier}`

## Ejemplos

### Ejemplo 1: Usando Tailwind para Layout y Espaciado

```html
<!-- ✅ Correcto - Usando utilidades de Tailwind -->
<div class="flex flex-col gap-4 p-6 bg-white rounded-lg shadow-md">
  <h2 class="text-2xl font-bold text-gray-900 mb-4">Sales List</h2>
  <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
    <app-sale-item [sale]="sale" />
  </div>
</div>

<!-- ❌ Incorrecto - CSS personalizado para utilidades comunes -->
<div class="sales-container">
  <h2 class="sales-title">Sales List</h2>
  <div class="sales-grid">
    <app-sale-item [sale]="sale" />
  </div>
</div>

<style>
.sales-container {
  display: flex;
  flex-direction: column;
  gap: 1rem;
  padding: 1.5rem;
  background: white;
  border-radius: 0.5rem;
  box-shadow: 0 4px 6px rgba(0, 0, 0, 0.1);
}
/* ... más CSS personalizado */
</style>
```

### Ejemplo 2: Combinando Tailwind con Clases BEM Personalizadas

```html
<!-- ✅ Correcto - Tailwind para layout, clase personalizada para estilos específicos del componente -->
<div class="flex items-center gap-4 p-4 bg-white rounded-lg">
  <div class="ft-user-card__avatar">
    <img [src]="user.avatar" [alt]="user.name" />
  </div>
  <div class="ft-user-card__content">
    <h3 class="text-lg font-semibold">{{ user.name }}</h3>
    <p class="text-gray-600">{{ user.email }}</p>
  </div>
</div>
```

```scss
// Estilos específicos del componente usando BEM
.ft-user-card__avatar {
  position: relative;
  width: 48px;
  height: 48px;
  border-radius: 50%;
  overflow: hidden;
  
  // Animación compleja que Tailwind no puede manejar fácilmente
  &::after {
    content: '';
    position: absolute;
    inset: 0;
    border: 2px solid transparent;
    border-radius: 50%;
    transition: border-color 0.3s ease;
  }
  
  &:hover::after {
    border-color: var(--primary-color);
  }
}

.ft-user-card__content {
  flex: 1;
  min-width: 0; // Soporte para truncamiento de texto
}
```

### Ejemplo 4: Componente con Tailwind y Estilos Personalizados

```typescript
@Component({
  selector: 'ft-button',
  template: `
    <button
      [class]="buttonClasses()"
      [disabled]="disabled()"
      (click)="handleClick()"
    >
      <ft-icon *ngIf="icon()" [name]="icon()" class="ft-button__icon" />
      <ng-content />
    </button>
  `,
  styles: [`
    .ft-button__icon {
      // Posicionamiento específico del icono del componente
      margin-right: 0.5rem;
      vertical-align: middle;
    }
    
    .ft-button--loading {
      // Animación de estado de carga
      position: relative;
      color: transparent;
      
      &::after {
        content: '';
        position: absolute;
        inset: 0;
        border: 2px solid currentColor;
        border-top-color: transparent;
        border-radius: 50%;
        animation: spin 0.6s linear infinite;
      }
    }
  `]
})
export class ButtonComponent {
  variant = input<'primary' | 'secondary' | 'danger'>('primary');
  size = input<'sm' | 'md' | 'lg'>('md');
  disabled = input<boolean>(false);
  loading = input<boolean>(false);
  icon = input<string | undefined>(undefined);
  
  buttonClasses = computed(() => {
    const base = 'inline-flex items-center justify-center font-medium rounded-lg transition-colors';
    const variants = {
      primary: 'bg-blue-600 text-white hover:bg-blue-700',
      secondary: 'bg-gray-200 text-gray-900 hover:bg-gray-300',
      danger: 'bg-red-600 text-white hover:bg-red-700',
    };
    const sizes = {
      sm: 'px-3 py-1.5 text-sm',
      md: 'px-4 py-2 text-base',
      lg: 'px-6 py-3 text-lg',
    };
    const state = this.disabled() ? 'opacity-50 cursor-not-allowed' : 'cursor-pointer';
    const loading = this.loading() ? 'ft-button--loading' : '';
    
    return `${base} ${variants[this.variant()]} ${sizes[this.size()]} ${state} ${loading}`;
  });
}
```

## Configuración

Tailwind CSS está configurado en `tailwind.config.js` con:
- Extensiones de tema personalizadas (colores, espaciado, tipografía)
- Breakpoints específicos del proyecto
- Clases de utilidad personalizadas si es necesario

```javascript
// ejemplo tailwind.config.js
module.exports = {
  content: [
    "./src/**/*.{html,ts}",
  ],
  theme: {
    extend: {
      colors: {
        primary: {
          DEFAULT: '#3b82f6',
          dark: '#2563eb',
        },
      },
    },
  },
  plugins: [],
}
```

## Consecuencias

### Positivas

- **Desarrollo Más Rápido:** Las clases de utilidad pre-construidas aceleran el desarrollo de UI
- **Consistencia:** Espaciado, colores y tamaños estandarizados en toda la aplicación
- **Tamaño de Bundle Más Pequeño:** Solo el CSS usado se incluye en el bundle final (con modo JIT)
- **Mantenibilidad:** No es necesario mantener CSS personalizado para patrones comunes
- **Diseño Responsivo:** Las utilidades responsivas integradas facilitan el desarrollo mobile-first
- **Experiencia del Desarrollador:** Menos cambio de contexto entre archivos HTML y CSS
- **Sistema de Diseño:** Tailwind proporciona una base sólida para un sistema de diseño

### Negativas

- **Curva de Aprendizaje:** Los miembros del equipo necesitan aprender los nombres de las clases de utilidad de Tailwind
- **Verbosidad HTML:** El HTML puede volverse más verboso con muchas clases de utilidad
- **Estilos Personalizados:** Algunos diseños complejos aún pueden requerir CSS personalizado
- **Configuración de Purge:** Necesidad de asegurar que Tailwind detecte correctamente todas las clases usadas

### Mitigación

- Documentación exhaustiva de Tailwind y hojas de referencia
- Extensiones de IDE para autocompletado de Tailwind
- Proceso de revisión de código para asegurar uso adecuado
- Guías de CSS personalizado claramente documentadas
- Entrenamiento regular del equipo en mejores prácticas de Tailwind

## Cuándo Usar CSS Personalizado

Usar CSS personalizado con el prefijo `ft-` cuando:
1. **Animaciones Complejas:** Animaciones multi-paso que son difíciles con Tailwind
2. **Lógica Específica del Componente:** Estilos que dependen del estado del componente de formas complejas
3. **Integración de Terceros:** Estilizar componentes de terceros que no pueden usar Tailwind
4. **Rendimiento:** Estilos críticos que necesitan optimizarse por separado
5. **Reutilización:** Estilos de componentes que se reutilizarán en toda la aplicación
