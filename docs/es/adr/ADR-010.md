# ADR-010: Estrategia de Uso de Iconos

**📖 [English](../../en/adr/ADR-010.md) | [Español](./ADR-010.md)**

**Estado:** Aceptado  
**Fecha:** 2025  
**Decisores:** Equipo de Arquitectura

## Contexto

Los iconos son una parte fundamental de la interfaz de usuario, y sin un enfoque estandarizado:
- Diferentes desarrolladores pueden usar diferentes bibliotecas de iconos o enfoques
- Renderizado de iconos inconsistente en toda la aplicación
- Difícil mantener y actualizar iconos
- No hay patrón claro para iconos personalizados
- Posibles problemas de rendimiento por múltiples bibliotecas de iconos
- Preocupaciones de accesibilidad con implementaciones de iconos inconsistentes

Necesitamos una estrategia consistente que:
- Estandarice el uso de iconos en toda la aplicación
- Proporcione un componente único para todo el renderizado de iconos
- Soporte tanto iconos predefinidos como personalizados
- Asegure accesibilidad y rendimiento
- Mantenga consistencia visual

## Decisión

Usaremos un **componente de iconos unificado** (`<ft-icon />`) que soporta múltiples colecciones de iconos e iconos personalizados. La implementación incluye:

1. **Componente de Iconos Único:** `<ft-icon />` para todo el renderizado de iconos
2. **Colecciones Predefinidas:** factoricons-slim, factoricons-regular, factoricons-solid
3. **Biblioteca de Iconos Personalizados:** Iconos personalizados almacenados en `public/images/icons.svg`
4. **Modificadores de Tamaño:** Clases de tamaño estandarizadas (`ft-icon--1` a `ft-icon--5`)
5. **Alineación con Material Design:** Todos los iconos se alinean con la grilla Material 24×24

## Implementación

### Uso del Componente de Iconos

Siempre usar el componente `<ft-icon />` para todos los iconos en la aplicación:

```html
<!-- ✅ Correcto - usando componente ft-icon -->
<ft-icon name="user" />
<ft-icon name="settings" collection="factoricons-slim" />
<ft-icon name="custom-icon" collection="icons" />

<!-- ❌ Incorrecto - usando otros enfoques de iconos -->
<mat-icon>person</mat-icon>
<svg>...</svg>
<i class="material-icons">person</i>
```

### Colecciones Disponibles

#### Colecciones Predefinidas

- **`factoricons-regular`** (por defecto) - No es necesario especificar la propiedad
- **`factoricons-slim`** - Variante delgada
- **`factoricons-solid`** - Variante sólida

#### Colección de Iconos Personalizados

- **`icons`** - Iconos personalizados almacenados en `public/images/icons.svg`

### Propiedades del Componente

El componente `<ft-icon />` acepta:

- **`name`** (requerido): El nombre del símbolo del icono
- **`collection`** (opcional): La fuente de la colección de iconos (por defecto `factoricons-regular`)

### Biblioteca de Iconos Personalizados

Cuando se usa la colección `icons`, los iconos deben existir en `public/images/icons.svg`.

#### Crear Iconos Personalizados

Cada icono personalizado debe definirse como un **símbolo SVG** optimizado a **24×24 píxeles**:

```html
<!-- public/images/icons.svg -->
<svg xmlns="http://www.w3.org/2000/svg" style="display: none;">
  <symbol id="settings" viewBox="0 0 24 24">
    <path fill="currentColor" d="M12 8a4 4 0 1 0 0 8a4 4 0 0 0 0-8z" />
  </symbol>
  
  <symbol id="user-settings" viewBox="0 0 24 24">
    <path fill="currentColor" d="M12 12a4 4 0 1 0 0-8a4 4 0 0 0 0 8z" />
  </symbol>
</svg>
```

**Requisitos:**
- Cada icono debe ser un elemento `<symbol>`
- El atributo `id` actúa como el nombre del icono (usado en la propiedad `name`)
- `viewBox` debe ser `0 0 24 24` para alineación con Material Design
- Usar `fill="currentColor"` para heredar el color del texto
- Optimizar paths para claridad y rendimiento

### Tamaños de Iconos

Para ajustar el tamaño del icono, usar la **clase modificadora de tamaño** `ft-icon--{n}` donde `{n}` va de **1 a 5**:

```html
<!-- ✅ Correcto - usando modificadores de tamaño -->
<ft-icon name="user" class="ft-icon--1" />  <!-- Extra pequeño -->
<ft-icon name="user" class="ft-icon--2" />  <!-- Pequeño -->
<ft-icon name="user" class="ft-icon--3" />  <!-- Mediano (por defecto) -->
<ft-icon name="user" class="ft-icon--4" />  <!-- Grande -->
<ft-icon name="user" class="ft-icon--5" />  <!-- Extra grande -->
```

**Escala de Tamaños:**
- `ft-icon--1`: Extra pequeño
- `ft-icon--2`: Pequeño
- `ft-icon--3`: Mediano (por defecto)
- `ft-icon--4`: Grande
- `ft-icon--5`: Extra grande

**Tamaños Personalizados:**

Si se requiere un tamaño específico más allá de la escala predefinida, aplicar un tamaño personalizado usando `font-size`:

```html
<!-- ✅ Correcto - tamaño personalizado -->
<ft-icon name="user" style="font-size: 32px;" />
```

### Ejemplos de Uso

#### Usando Colección por Defecto (factoricons-regular)

```html
<ft-icon name="user" />
<ft-icon name="settings" />
<ft-icon name="home" />
```

#### Usando Otra Colección Predefinida

```html
<ft-icon name="user" collection="factoricons-slim" />
<ft-icon name="settings" collection="factoricons-solid" />
```

#### Usando Iconos Personalizados

```html
<ft-icon name="user-settings" collection="icons" />
<ft-icon name="calendar-add" collection="icons" />
```

#### Con Modificadores de Tamaño

```html
<ft-icon name="user" class="ft-icon--2" />
<ft-icon name="settings" class="ft-icon--4" />
```

#### En Componentes

```html
<button mat-flat-button color="primary">
  <ft-icon name="save" class="ft-icon--2" />
  <span i18n>Save</span>
</button>

<mat-form-field appearance="outline">
  <mat-label i18n>Search</mat-label>
  <ft-icon name="search" matPrefix class="ft-icon--2" />
  <input matInput type="text" />
</mat-form-field>
```

## Mejores Prácticas

### 1. Preferir Colecciones Predefinidas

```html
<!-- ✅ Correcto - usar colecciones predefinidas cuando sea posible -->
<ft-icon name="user" />
<ft-icon name="settings" collection="factoricons-slim" />

<!-- ❌ Incorrecto - crear icono personalizado cuando existe predefinido -->
<ft-icon name="user-icon" collection="icons" />
```

### 2. Solo Agregar Iconos Personalizados Cuando sea Necesario

- Solo agregar nuevos iconos a la colección `icons` cuando no existe un símbolo adecuado en los conjuntos predefinidos
- Buscar primero dentro de la colección deseada antes de crear iconos personalizados
- Los iconos personalizados deben seguir las guías de la grilla Material Design 24×24

### 3. Usar Modificadores de Tamaño Consistentemente

```html
<!-- ✅ Correcto - usar modificadores de tamaño estándar -->
<ft-icon name="user" class="ft-icon--3" />

<!-- ❌ Incorrecto - estilos inline para tamaños estándar -->
<ft-icon name="user" style="font-size: 24px;" />
```

### 4. Mantener Consistencia Visual

- Todos los iconos (personalizados o predefinidos) deben alinearse visualmente con la **grilla Material 24×24** para consistencia
- Usar `fill="currentColor"` en símbolos SVG personalizados para heredar el color del texto
- Asegurar que los iconos estén optimizados y claros en todos los tamaños

### 5. Guías de Selección de Iconos

- Para seleccionar un icono, **buscar dentro de la colección deseada** un símbolo cuyo `id` mejor represente el significado o concepto deseado
- El `id` de cada `<symbol>` actúa como el **identificador descriptivo** y se usa en la propiedad `name` del componente `<ft-icon />`
- Elegir nombres de iconos que sean claros y descriptivos

## Convenciones de Nombres de Iconos

### Colecciones Predefinidas

- Usar kebab-case para nombres de iconos
- Los nombres deben ser descriptivos y claros
- Ejemplos: `user`, `settings`, `home`, `arrow-right`, `check-circle`

### Iconos Personalizados

- Usar kebab-case para IDs de iconos personalizados
- Usar nombres compuestos descriptivos cuando sea necesario
- Ejemplos: `user-settings`, `calendar-add`, `file-upload`

## Accesibilidad

- Los iconos deben ir acompañados de etiquetas de texto cuando se usan como elementos interactivos
- Usar `aria-label` o `aria-labelledby` para botones solo con iconos
- Asegurar contraste de color suficiente para los iconos
- Los iconos deben ser reconocibles y claros en todos los tamaños

## Consecuencias

### Positivas

- **Consistencia:** Todos los iconos usan el mismo componente y enfoque
- **Mantenibilidad:** Fuente única de verdad para el renderizado de iconos
- **Rendimiento:** Carga y renderizado de iconos optimizados
- **Flexibilidad:** Soporte para iconos predefinidos y personalizados
- **Accesibilidad:** Patrones de accesibilidad consistentes
- **Consistencia Visual:** Todos los iconos se alinean con la grilla Material Design

### Negativas

- **Curva de Aprendizaje:** Los miembros del equipo necesitan aprender la API del componente de iconos
- **Creación de Iconos Personalizados:** Requiere conocimiento de SVG para iconos personalizados
- **Gestión de Archivos:** Los iconos personalizados requieren mantener el archivo `icons.svg`

### Mitigación

- Documentación y ejemplos claros
- Proceso de revisión de código para asegurar uso adecuado
- Herramientas de búsqueda de bibliotecas de iconos y documentación
- Revisión regular de iconos personalizados para asegurar que aún sean necesarios
