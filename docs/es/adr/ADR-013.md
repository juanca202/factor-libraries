# ADR-013: Layout y Estructura de Formularios

**📖 [English](../../en/adr/ADR-013.md) | [Español](./ADR-013.md)**

**Estado:** Aceptado  
**Fecha:** 2025  
**Decisores:** Equipo de Arquitectura

## Contexto

El layout y la estructura de los formularios impactan directamente la experiencia del usuario y la mantenibilidad del código. Sin patrones de layout estandarizados:

- Los formularios tienen espaciado y alineación inconsistentes
- El comportamiento responsive es impredecible
- Aumenta la duplicación de código cuando se recrean layouts similares
- La jerarquía visual no es clara
- El mantenimiento se vuelve difícil cuando los patrones de layout varían

Necesitamos un enfoque consistente que:
- Proporcione layouts de formularios predecibles y responsivos
- Asegure consistencia visual en toda la aplicación
- Haga los formularios fáciles de mantener y modificar
- Funcione perfectamente con componentes de Angular Material
- Siga las mejores prácticas modernas de CSS Grid

## Decisión

Usaremos **CSS Grid** con clases de utilidad de Tailwind CSS para todos los layouts de formularios, siguiendo estos patrones estandarizados:

1. **Contenedor Grid:** Todos los campos del formulario deben estar contenidos en un `div` con la clase `grid`
2. **Configuración de Columnas:** Usar `grid-cols-{n}` para definir el número de columnas según el contenido
3. **Campos de Ancho Completo:** Los campos que abarquen todo el ancho usan `col-span-{n}` donde `n` coincide con el número de columnas
4. **Espaciado:** Espaciado estandarizado usando `gap-x-4 gap-y-2`
5. **Campos de Formulario:** `mat-form-field` simple sin atributos de apariencia ni clases de ancho

## Implementación

### Estructura del Contenedor Grid

Todos los campos del formulario deben estar envueltos en un contenedor grid:

```html
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <!-- Campos del formulario aquí -->
</div>
```

### Configuración de Columnas

Elige el número de columnas según el contenido y los requisitos de diseño del formulario:

- **2 columnas:** Más común para formularios con pares de campos relacionados (ej: nombre + apellido)
- **3 columnas:** Para formularios con tríos de campos relacionados
- **1 columna:** Para formularios simples o cuando todos los campos deben ser de ancho completo

```html
<!-- Grid de 2 columnas (más común) -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>...</mat-form-field>
  <mat-form-field>...</mat-form-field>
</div>

<!-- Grid de 3 columnas -->
<div class="grid grid-cols-3 gap-y-2 gap-x-4">
  <mat-form-field>...</mat-form-field>
  <mat-form-field>...</mat-form-field>
  <mat-form-field>...</mat-form-field>
</div>

<!-- Grid de 1 columna (campos de ancho completo) -->
<div class="grid grid-cols-1 gap-y-2 gap-x-4">
  <mat-form-field>...</mat-form-field>
</div>
```

### Campos de Ancho Completo

Los campos que deben abarcar todo el ancho del grid usan `col-span-{n}` donde `n` coincide con el número de columnas:

```html
<!-- Grid de 2 columnas con campos de ancho completo -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>
    <mat-label i18n>Nombre</mat-label>
    <input matInput formControlName="firstName" />
  </mat-form-field>
  <mat-form-field>
    <mat-label i18n>Apellido</mat-label>
    <input matInput formControlName="lastName" />
  </mat-form-field>
  <mat-form-field class="col-span-2">
    <mat-label i18n>Email</mat-label>
    <input matInput type="email" formControlName="email" />
  </mat-form-field>
  <mat-form-field class="col-span-2">
    <mat-label i18n>Teléfono</mat-label>
    <input matInput formControlName="phone" />
  </mat-form-field>
</div>
```

### Espaciado

El espaciado estandarizado asegura un ritmo visual consistente:

- **Espaciado horizontal:** `gap-x-4` (1rem / 16px)
- **Espaciado vertical:** `gap-y-2` (0.5rem / 8px)

```html
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <!-- Campos espaciados automáticamente -->
</div>
```

### Estructura de Campos de Formulario

Los campos de formulario deben ser simples y limpios:

- **Sin atributo appearance:** No usar `appearance="outline"` u otras variantes de apariencia
- **Sin clases de ancho:** No usar `class="ft-w-100"` u otras utilidades de ancho similares
- **Estructura simple:** Dejar que el grid maneje el layout y el espaciado

```html
<!-- ✅ Correcto -->
<mat-form-field>
  <mat-label i18n>Etiqueta del Campo</mat-label>
  <input matInput formControlName="fieldName" />
  @if (form.get('fieldName')?.invalid && form.get('fieldName')?.touched) {
    <mat-error>{{ form.get('fieldName') | errorMessage }}</mat-error>
  }
</mat-form-field>

<!-- ❌ Incorrecto -->
<mat-form-field appearance="outline" class="ft-w-100">
  <mat-label i18n>Etiqueta del Campo</mat-label>
  <input matInput formControlName="fieldName" />
</mat-form-field>
```

## Ejemplo Completo de Formulario

```html
<form [formGroup]="form" (ngSubmit)="onSubmit()">
  <div mat-dialog-content>
    <div class="ft-container">
      <div class="grid grid-cols-2 gap-y-2 gap-x-4">
        <mat-form-field>
          <mat-label i18n>Nombre</mat-label>
          <input matInput formControlName="firstName" />
          @if (form.get('firstName')?.invalid && form.get('firstName')?.touched) {
            <mat-error>{{ form.get('firstName') | errorMessage }}</mat-error>
          }
        </mat-form-field>
        
        <mat-form-field>
          <mat-label i18n>Apellido</mat-label>
          <input matInput formControlName="lastName" />
          @if (form.get('lastName')?.invalid && form.get('lastName')?.touched) {
            <mat-error>{{ form.get('lastName') | errorMessage }}</mat-error>
          }
        </mat-form-field>
        
        <mat-form-field class="col-span-2">
          <mat-label i18n>Email</mat-label>
          <input matInput type="email" formControlName="email" />
          @if (form.get('email')?.invalid && form.get('email')?.touched) {
            <mat-error>{{ form.get('email') | errorMessage }}</mat-error>
          }
        </mat-form-field>
        
        <mat-form-field class="col-span-2">
          <mat-label i18n>Teléfono</mat-label>
          <input matInput formControlName="phone" />
          @if (form.get('phone')?.invalid && form.get('phone')?.touched) {
            <mat-error>{{ form.get('phone') | errorMessage }}</mat-error>
          }
        </mat-form-field>
      </div>
    </div>
  </div>
  
  <div mat-dialog-actions>
    <button type="button" mat-button mat-dialog-close i18n>Cancelar</button>
    <button type="submit" mat-flat-button color="primary" [disabled]="form.invalid" i18n>
      Guardar
    </button>
  </div>
</form>
```

## Mejores Prácticas

### 1. Elegir el Número de Columnas Apropiado

```html
<!-- ✅ Correcto - 2 columnas para campos de nombre -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>Nombre</mat-form-field>
  <mat-form-field>Apellido</mat-form-field>
</div>

<!-- ✅ Correcto - 1 columna para campo único o todos de ancho completo -->
<div class="grid grid-cols-1 gap-y-2 gap-x-4">
  <mat-form-field>Descripción</mat-form-field>
</div>
```

### 2. Usar col-span para Campos de Ancho Completo

```html
<!-- ✅ Correcto -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>Campo 1</mat-form-field>
  <mat-form-field>Campo 2</mat-form-field>
  <mat-form-field class="col-span-2">Campo de ancho completo</mat-form-field>
</div>

<!-- ❌ Incorrecto - no usar grid separado o clases de ancho -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>Campo 1</mat-form-field>
  <mat-form-field>Campo 2</mat-form-field>
</div>
<div class="grid grid-cols-1">
  <mat-form-field class="ft-w-100">Campo de ancho completo</mat-form-field>
</div>
```

### 3. Mantener los Campos de Formulario Simples

```html
<!-- ✅ Correcto - estructura simple y limpia -->
<mat-form-field>
  <mat-label i18n>Etiqueta</mat-label>
  <input matInput formControlName="field" />
</mat-form-field>

<!-- ❌ Incorrecto - atributos y clases innecesarias -->
<mat-form-field appearance="outline" class="ft-w-100">
  <mat-label i18n>Etiqueta</mat-label>
  <input matInput formControlName="field" />
</mat-form-field>
```

### 4. Espaciado Consistente

```html
<!-- ✅ Correcto - siempre usar gap-x-4 gap-y-2 -->
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <!-- Campos -->
</div>

<!-- ❌ Incorrecto - espaciado inconsistente -->
<div class="grid grid-cols-2 gap-4">
  <!-- Campos -->
</div>
```

## Consideraciones Responsive

Para formularios responsivos, considera usar los breakpoints responsive de Tailwind:

```html
<!-- 1 columna en móvil, 2 columnas en pantallas más grandes -->
<div class="grid grid-cols-1 md:grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>Campo 1</mat-form-field>
  <mat-form-field>Campo 2</mat-form-field>
  <mat-form-field class="md:col-span-2">Ancho completo en escritorio</mat-form-field>
</div>
```

## Integración con ADR-008

Este ADR complementa [ADR-008: Estrategia de Validación de Formularios](./ADR-008.md):

- **ADR-008** define patrones de validación y manejo de errores
- **ADR-013** define patrones de layout y estructura
- Ambos deben usarse juntos para una implementación completa de formularios

Ejemplo combinando ambos ADRs:

```html
<div class="grid grid-cols-2 gap-y-2 gap-x-4">
  <mat-form-field>
    <mat-label i18n>Email</mat-label>
    <input matInput type="email" formControlName="email" />
    @if (form.get('email')?.invalid && form.get('email')?.touched) {
      <mat-error>{{ form.get('email') | errorMessage }}</mat-error>
    }
  </mat-form-field>
</div>
```

## Consecuencias

### Positivas

- **Consistencia:** Todos los formularios siguen el mismo patrón de layout
- **Mantenibilidad:** Fácil modificar y actualizar layouts de formularios
- **Responsividad:** El sistema de grid proporciona capacidades responsive integradas
- **Armonía Visual:** El espaciado consistente crea mejor ritmo visual
- **Claridad del Código:** La estructura clara hace los formularios más fáciles de leer y entender
- **Flexibilidad:** El sistema de grid permite varias configuraciones de columnas

### Negativas

- **Curva de Aprendizaje:** Los miembros del equipo necesitan aprender el patrón de grid
- **Rigidez:** Puede no encajar en todos los casos extremos (aunque el grid es muy flexible)
- **Dependencia de Tailwind:** Requiere que Tailwind CSS esté disponible

### Mitigación

- Documentación exhaustiva en este ADR
- Ejemplos de código y plantillas
- Proceso de revisión de código para asegurar cumplimiento del patrón
- Guías claras sobre cuándo usar diferentes números de columnas
